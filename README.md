# 🧠 Neural Maze Trainer

Интерактивное веб-приложение, в котором нейронная сеть обучается проходить лабиринт двумя методами: **генетическим алгоритмом** и **обучением с подкреплением (Q-learning)**.

---

## 🚀 Как запустить

Просто откройте `index.html` в современном браузере (Chrome, Firefox, Edge ≥ 90).  
Никаких сборщиков и серверов не нужно — всё работает на нативных ES-модулях.

```
git clone https://github.com/berlandbors/LearnNetworkNeuron-0.git
cd LearnNetworkNeuron-0
# открыть index.html в браузере
```

> **Внимание:** из-за политики CORS ES-модули не работают напрямую через `file://`.  
> Используйте любой локальный HTTP-сервер:
> ```bash
> npx serve .
> # или
> python -m http.server 8080
> ```

## 🧠 Нейронная сеть

| Слой   | Размер | Активация |
|--------|--------|-----------|
| Вход   | 24     | —         |
| Скрытый | 8–64  | ReLU      |
| Выход  | 4      | Sigmoid   |

**Входной вектор (24):**
- 8 расстояний до стен (кардинальные + диагонали)
- Нормализованная позиция агента (x, y)
- Нормализованная позиция цели (x, y)
- Угол к цели (cos, sin)
- Расстояние до цели
- Последние 4 действия (one-hot)
- Стены по 4 направлениям (0/1)
- Нормализованное число шагов

**Выход (4):** вверх / вниз / влево / вправо

## 🧬 Генетический алгоритм

1. Инициализация случайной популяции N агентов
2. Каждый агент проходит эпизод в лабиринте
3. Вычисляется fitness = накопленные награды + бонус за достижение цели
4. **Турнирная селекция** (k=5) выбирает родителей
5. **Равномерный кроссовер** смешивает веса двух родителей
6. **Мутация** с вероятностью `rate` изменяет каждый вес
7. **Элитизм** — 2 лучших агента переходят без изменений
8. Повтор с шага 2

---

## 🤖 Q-обучение (RL)

- **ε-greedy**: сначала случайные действия (ε=1.0), затем эксплуатация знаний (ε→0.05)
- **Experience Replay**: агент сохраняет опыт в буфере (5000 примеров), обучается на случайных мини-батчах (32)
- **Целевая сеть**: отдельная копия сети для вычисления Q-целей (обновляется каждые 100 шагов)
- **Дисконтирование**: γ = 0.95

## 🏆 Система наград

| Событие | Награда |
|---------|---------|
| Достижение цели | +1000 |
| Приближение к цели | +10 × Δdist |
| Удаление от цели | +5 × Δdist (отриц.) |
| Столкновение со стеной | −10 |
| Каждый шаг | −0.1 |
| Посещение уже посещённой клетки | −2 |

---

## 🗺️ Алгоритм генерации лабиринтов

Используется **Recursive Backtracker (DFS)**:
1. Начать из случайной клетки
2. Случайно выбрать непосещённого соседа
3. Убрать стену между текущей клеткой и соседом
4. Перейти в соседа и повторить
5. При тупике — вернуться (backtrack)

Результат: идеальный лабиринт (ровно один путь между любыми двумя точками).
